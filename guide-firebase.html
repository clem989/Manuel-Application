Synchroniser avec Firebase

≈ 10 minutes, une seule fois. À la fin : tes notes, lectures, habitudes et stats se synchronisent en temps réel entre ton téléphone et ton PC Linux. 100 % gratuit (plan « Spark » de Google, largement suffisant pour un usage personnel).

01

Créer le projet

  1. Va sur console.firebase.google.com et connecte-toi avec ton compte Google.
  2. Clique Créer un projet → nom : manuel (ou ce que tu veux).
  3. Quand on te propose Google Analytics : désactive-le (inutile ici), puis Créer le projet.
02

Activer Firestore (la base de données)

  1. Menu de gauche → Créer (Build) → Firestore Database → Créer une base de données.
  2. Emplacement : choisis europe-west1 (Belgique) ou eur3.
  3. Mode : production (on met les règles juste après) → Créer.
  4. Une fois créée, onglet Règles : remplace tout le contenu par ceci, puis Publier :
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{uid}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == uid;
    }
  }
}
Ce que ça veut direSeul TON compte Google peut lire et écrire TES données. Personne d'autre, même en connaissant l'adresse du projet.
03

Activer la connexion Google

  1. Menu de gauche → Créer → Authentication → Commencer.
  2. Onglet Sign-in method → Google → interrupteur Activer.
  3. Choisis ton adresse e-mail comme « adresse d'assistance » → Enregistrer.
04

Autoriser ton site Netlify

  1. Toujours dans Authentication → onglet Settings (Paramètres) → Domaines autorisés.
  2. Clique Ajouter un domaine et saisis ton adresse Netlify, par exemple manuel-clement-xyz.netlify.app (sans https://).
ImportantSans cette étape, le bouton « Se connecter » échouera avec une erreur auth/unauthorized-domain. Si tu changes un jour d'URL Netlify, reviens ajouter la nouvelle ici.
05

Récupérer la configuration

  1. Roue dentée ⚙️ (en haut à gauche, à côté de « Vue d'ensemble ») → Paramètres du projet.
  2. Descends jusqu'à Vos applications → clique l'icône Web </>.
  3. Surnom : manuel → ne coche PAS « Firebase Hosting » → Enregistrer l'application.
  4. Un bloc de code apparaît, contenant const firebaseConfig = { apiKey: "...", ... }. Copie tout ce bloc (le bouton copier de la console fait l'affaire).
Pas un secretCette « clé API » identifie ton projet, elle ne donne aucun accès à tes données : ce sont les règles de l'étape 02 qui protègent. Tu peux la coller sans crainte.
06

Brancher l'app (sur chaque appareil)

  1. Ouvre Manuel → onglet Stats → carte Synchronisation → Configurer.
  2. Colle le bloc copié à l'étape 05 → Enregistrer.
  3. Le bouton devient Se connecter : touche-le et choisis ton compte Google.
  4. Le statut passe à ✅ Synchronisé. Refais les mêmes gestes (coller + connexion) sur ton PC Linux : 2 minutes.
VérificationAjoute une note sur le téléphone : elle apparaît sur le PC en 1 à 3 secondes (et inversement). Hors ligne ? Les changements partent tout seuls au retour du réseau.
07

Bon à savoir

Gratuit, vraiment ? Oui : le plan Spark offre 50 000 lectures et 20 000 écritures par jour. Ton usage personnel en consommera quelques dizaines.

Qu'est-ce qui se synchronise ? Tout : notes, lectures, citations, latin, alexandrins, tâches, objectifs, habitudes, rappels, stats, Pomodoros, réglages du mode nuit. La fusion se fait élément par élément : la version la plus récente gagne, rien ne s'écrase bêtement.

Et la sauvegarde JSON ? Elle reste utile comme ceinture de sécurité (Stats → Sauvegarde) — un export de temps en temps ne coûte rien.

Déconnexion : le bouton « Déconnexion » arrête la synchro mais ne supprime rien, ni sur l'appareil ni dans le cloud.
